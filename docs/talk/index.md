# Where to talk?

Communication is a critical part of software development. This section covers tools for
documentation, writing, and presentation.

## Topics

- **[Markdown](markdown.md)** – Lightweight markup language
- **[LaTeX & Overleaf](latex-overleaf.md)** – Typesetting system and online editor

## Philosophy

Your communication tools should:

- Be expressive (convey complex ideas clearly)
- Be portable (work across platforms)
- Support collaboration (multiple authors)
- Produce professional results

## Getting Started

1. Master Markdown for everyday documentation
2. Learn LaTeX for academic and formal documents
3. Use Overleaf for collaborative LaTeX editing
4. Choose the right tool for each communication task

## Pro Tips

- **Write in Markdown first**: Convert to other formats as needed
- **Learn LaTeX for precision**: When formatting matters
- **Use version control for documents**: Git works for text documents too
- **Automate document generation**: Generate reports from data
