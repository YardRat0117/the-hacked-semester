# Where to deliver?

Building, testing, and debugging are essential parts of the development process. This section covers
tools for delivering working software.

## Topics

- **[GNU Make](gnu-make.md)** – Build automation tool
- **[CMake](cmake.md)** – Cross-platform build system
- **[GDB](gdb.md)** – GNU Debugger for debugging programs

## Philosophy

Your build and debug tools should:

- Be reliable (produce consistent results)
- Be fast (minimize waiting time)
- Provide good feedback (help you understand what went wrong)
- Integrate with your workflow

## Getting Started

1. Learn Make for simple build automation
2. Use CMake for complex, cross-platform projects
3. Master debugging with GDB
4. Integrate these tools into your development process

## Pro Tips

- **Automate builds**: Never compile manually
- **Write good Makefiles**: Clear, maintainable, and efficient
- **Learn debugging techniques**: Print statements are good, but debuggers are better
- **Profile your builds**: Identify and fix slow compilation steps
