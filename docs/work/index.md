# Where to work?

Your tools determine how efficiently you can write, debug, and collaborate on code. This section
covers essential development tools.

## Topics

- **[Vim](vim.md)** – A powerful, modal text editor
- **[LSP](lsp.md)** – Language Server Protocol for IDE features
- **[Git](git.md)** – Version control system
- **[GitHub & GitLab](github-gitlab.md)** – Collaboration platforms

## Philosophy

Your tools should:

- Reduce friction (not add to it)
- Be extensible (adapt to your needs)
- Support collaboration (work well with others)
- Have a learning curve worth climbing

## Getting Started

1. Choose an editor and learn it deeply
2. Set up LSP for your languages
3. Master Git fundamentals
4. Learn platform-specific workflows

## Pro Tips

- **Learn keyboard shortcuts**: They save thousands of clicks
- **Customize your editor**: Make it work for your specific needs
- **Write good commit messages**: They're documentation for your future self
- **Use Git effectively**: Branches, rebasing, and collaboration patterns
