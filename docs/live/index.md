# Where to live?

Your development environment is your home as a programmer. This section covers everything you need
to set up a comfortable, efficient workspace.

## Topics

- **[Terminal & Shell](terminal-shell.md)** – Master the command line interface
- **[WSL](wsl.md)** – Windows Subsystem for Linux for Windows users
- **[Linux Distros](linux-distros.md)** – Choosing and using Linux distributions
- **[SSH](ssh.md)** – Secure remote access and development

## Philosophy

Your development environment should:

- Be reproducible (you should be able to recreate it easily)
- Be portable (work across machines)
- Fit your workflow (not force you into someone else's)
- Be maintainable (easy to update and backup)

## Getting Started

1. Start with the terminal – it's the foundation
2. Choose an OS environment that works for you
3. Learn SSH for remote development
4. Automate setup with scripts (see `scripts/` directory)

## Pro Tips

- **Document your setup**: Keep a record of your configuration
- **Version control your dotfiles**: Use Git to manage configuration files
- **Automate everything**: Write scripts for repetitive setup tasks
- **Test your setup**: Regularly test that you can recreate your environment
